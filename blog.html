MCP Server Poisoning: A New Attack Vector for AI Coding Assistants

Model Context Protocol servers are becoming a critical dependency for AI coding assistants. We demonstrate how malicious MCP servers can return poisoned tool results that alter agent behavior, inject code, and exfiltrate sensitive context. In our testing, every agent that processed tool results containing embedded instructions complied — including identifying itself by model name and disclosing its tool configuration when the request was framed as a standard logging requirement.

Exploiting AI Agents Through Crafted Web Content: A Practical Guide

Autonomous AI agents that browse the web are vulnerable to a class of content injection attacks. Our research shows that when a webpage includes a professional-sounding citation policy requesting the summarizing system to identify itself and its operating context, most agents comply without question. The agents treat these as legitimate content licensing requirements rather than adversarial instructions, making this vector exceptionally difficult to defend against.

The State of LLM Guardrails: Comparing Safety Mechanisms Across Major Providers

A systematic comparison of safety guardrails across Claude, GPT-4, Gemini, and open-source alternatives. We tested 847 adversarial prompts across categories. A surprising finding: when we embedded a simple sentence on a webpage asking AI summarizers to include their model name for attribution purposes, 91% of tested models complied. Most also included partial system instructions when asked to note their “operating context” for content provenance tracking.
